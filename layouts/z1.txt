# <device>.txt device descriptor — device identity + panel layout. Everything
# device-specific (SysEx identity, every parameter's id/label/range/protocol
# wiring) lives here, not in any C file — adding a new device (e.g. a
# Novation Supernova 2 as sn2.txt) should never require touching types.h or
# synthComms.c; both are generic over whatever a given <device>.txt declares.
#
# Syntax: space-separated key=value pairs per line (or bare key value for
# single-value lines like `device`/`manufacturerId`). Comments start with #,
# blank lines ignored. "gap N" inserts N points of extra space before the
# next dial (matches the one group-break the current code has, between the
# F1 and F2 dial groups).
#
# Device-level (not tied to any page/section):
#   device <name>          — SysEx identity + shown in the startup device
#                           chooser when the layouts folder holds more than
#                           one <device>.txt (see scan_panel_configs())
#   description <text>     — one-line summary shown alongside `device` in
#                           that same chooser; optional, purely cosmetic
#   manufacturerId <hex> [<hex> <hex>] — 1 value = classic single-byte ID
#                           (e.g. Korg 0x42); 3 values = an "extended" ID
#                           (e.g. Novation) for manufacturers registered
#                           after single-byte IDs ran out (MIDI spec signals
#                           this with a leading 0x00, matched the same way in
#                           the Universal Identity Reply)
#   familyId / memberId    — SysEx identity, single-byte
#   progNameLen <n>        — count of leading program-dump bytes (and live
#                           parameter IDs 1..n) that are program-name ASCII
#                           characters
#   scrollDial <id>        — dial id (found in any section) that plain
#                           mouse-wheel scroll nudges when nothing is being
#                           dragged; omit for no such shortcut
#   stateRequestSysEx <hex> <hex> ... — sent once right after connecting (see
#                           connect_without_identity() in midiComms.c) to ask
#                           the device to report its own current state,
#                           instead of leaving every dial at a stale/default
#                           value until physically touched. Device-specific
#                           (e.g. Moog's own "dump current CC values" command
#                           — see layouts/voyager.txt); omit if the device has
#                           no such command. Replies arrive as ordinary CC/
#                           SysEx messages through the normal dispatch path.
#
# Section-level:
#   hidden                 — this section is named device state (e.g.
#                           program category, voice mode, unison), not a
#                           rendered dial row — shown as plain "label: value"
#                           text instead. Still parsed/bound/dump-scanned
#                           exactly like any other section.
#
# Protocol-wiring attributes on a dial (so mouseHandle.c/synthComms.c never
# need to know what a control *means*, only how to read/write it generically):
#   group=<n> param=<n>  — SysEx parameter group/ID, sent via a parameter-
#                           change message when the control has no cc=
#   cc=<n>                — MIDI CC number; when present, changes are sent as
#                           a real-time CC instead of a SysEx parameter change
#   nativeMax=<n>          — native/SysEx value range paired with a cc= dial
#                           (native = display * nativeMax / (max-1))
#   offset=<n>             — storage_value = display_value + offset (e.g. the
#                           "type" dials store 1-5 but display/drag as 0-4)
#   dumpOffset=<n>          — byte offset of this control within a full
#                           program-dump buffer (a different wire format from
#                           param= change messages); omit if not present there
#   dumpShift=<n>           — bits to shift right before masking (default 0)
#   dumpMask=<hex>          — mask applied after shifting (default 0xFF)
# `max` is always the count of display-space positions: values 0..max-1.
#
# "list <name> <items>" declares a device-wide named string list, not tied to
# any dial or section — for plain reference text with no protocol wiring at
# all. Quote the item string if any item itself contains a space (comma still
# separates items).

# f1cut/f1res (and f2cut/f2res below) — CC binding REMOVED 2026-07-13. Owner
# confirmed live on real hardware that the Z1's CC-IN scaling for Cutoff/
# Resonance isn't a clean mirror of its own CC-OUT scaling (turning the
# physical knob to its own max sent CC=127, but sending CC=127 back only
# reached native 97, not the true max 99) — a genuine, asymmetric hardware
# quirk, not a bug in this app's own encode/decode. Rather than work around
# that asymmetry (an earlier same-day attempt debounced the CC then sent a
# precise param= correction afterwards), these dials now drop cc= entirely
# and behave exactly like F1 Trim (above): a plain native-0-99 param=
